# 任务赏金支付对接说明

任务模块已对接微信支付：发布者在「待确认」时点击「确认完成」会先调起赏金支付，支付成功后由微信回调云函数将任务置为已完成。

---

## 一、流程说明

1. 接单人提交完成 → 任务变为「待确认」。
2. 发布者点击「确认完成」→ 弹窗提示支付赏金 → 调云函数 `task` 的 `createTaskPayment` 创建支付单。
3. 小程序调起 `wx.requestPayment`，用户完成支付。
4. 微信支付后台异步调用云函数 `taskPayCallback`，云函数将对应任务更新为「已完成」并写入 `rewardPaidAt`。
5. 若未配置支付（无商户号），则提示「未开通在线支付」，可选择「线下支付后确认完成」，仍走原 `confirmComplete` 逻辑。

---

## 二、商户与云开发配置（必做）

### 1. 微信支付商户号

- 在 [微信支付商户平台](https://pay.weixin.qq.com/) 注册并开通商户号。
- 将商户号与小程序绑定（在商户平台或公众平台「微信支付」中完成）。

### 2. 云开发开通云支付

- 登录 [微信公众平台](https://mp.weixin.qq.com) → 进入你的小程序。
- 打开 **云开发** → 选择对应环境 → 在「设置」或「能力」中开通 **云支付**（或「微信支付」），并按指引完成商户号关联（会得到用于云函数的**子商户号 sub_mch_id**）。

### 3. 云函数环境变量

在 **云开发控制台** → **云函数** 中，为云函数 **task** 配置环境变量：

| 变量名 | 说明 | 示例 |
|--------|------|------|
| `PAY_SUB_MCH_ID` | 微信支付子商户号（云支付关联后获得的 sub_mch_id） | `1900009231` |
| `ENV_ID` | 云开发环境 ID（与小程序使用的 env 一致，可选；不填时由小程序传入） | `intelligence-communi-4bcfec6c3b1` |

若不在云函数中配置 `ENV_ID`，则需在小程序侧传入：`config/index.js` 中的 `cloudEnvId` 需与云开发环境 ID 一致（已与 `app.js` 中 `wx.cloud.init` 的 `env` 保持一致即可）。

### 4. 部署云函数

- 上传并部署 **task**（含 `createTaskPayment`）。
- 上传并部署 **taskPayCallback**（支付结果回调，必须部署后微信才能回调成功）。

---

## 三、小程序侧配置

- **config/index.js** 中的 `cloudEnvId` 已与当前云环境一致，用于创建支付单时传给云函数；若更换云环境，请同步修改 `cloudEnvId` 和 `app.js` 中的 `env`。

---

## 四、数据库

- 任务文档在支付回调成功后会增加字段：`rewardPaidAt`（赏金支付完成时间）。
- 无需新增集合；如需对账，可自行在回调中写入 `task_payments` 等集合。

---

## 五、未配置支付时的行为

- 未设置 `PAY_SUB_MCH_ID` 或创建支付单失败时，会提示「未开通在线支付」。
- 用户可选择「线下支付后确认完成」，仍会调用 `confirmComplete`，任务照常变为已完成，仅无线上支付记录。

---

## 六、测试建议

1. 在开发/体验版中发布任务并走完流程至「待确认」。
2. 使用已绑定商户号的微信号点击「确认完成」，应调起支付。
3. 支付成功后，任务应变为「已完成」，且详情中可看到完成时间（及后续可扩展的 `rewardPaidAt`）。
4. 在云开发控制台查看 **taskPayCallback** 的调用日志，确认微信回调已触发且无报错。
